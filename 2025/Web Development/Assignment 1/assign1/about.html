About this Assignment

What special features have you done or attempted in creating the website?

  • Implemted server-side validation for status code format and duplicate verification to ensure status codes are uniquely indetifiable and formatted in a standardized manner that limits human/input error.
  • Built keyword search with case-insensitive matching to ensure search input handles both mixed and non-mixed case lettering.
  • Implemented a count feature in status search results to enhance readability of larger keyword matching results.

Which part(s) did you have trouble with?

  • Handling date parsing in dd/mm/yyyy and converting to MySQL's YYYY-MM-DD format within post status functionality, then converting from MySQL format (YYYY-MM-DD) to e.g. September 3, 2026 within the search status functionality.
  • Handling the format specifications within the post status page; even when inputting correctly formatted status an error message would appear that the format is invalid - this was due to incorrect parameters within poststatusform.php when defining the input.

What would you like to do better next-time?

  • Improve modularity of the code to enhance scalability prospects by avoiding repetitive logic.
  • Enhance user-experience by incorporating human-computer interaction principles, user research, and other HCI components to optimize the functionality of the website.
  • Reframe logic to handle larger datasets in more sensical ways; for example: incorporating a ranking system within the search function to return the most relevant results at the top, instead of the current state by which it just orders the returned search results by date.

What have you learnt along the way? Did you use any references or resources during this project? if so, please incliude the sources.

  • How to effectively handle date formatting in conjunction with input forms and MySQL database handling.
  • How to detect table existence or search queries parsed through PHP code to MySQL databases
  • Resources used include the PHP Manual for PHP documentation and fundamental understanding, GeeksforGeeks.org for quick and simple PHP, MySQL, and HTML breakdowns of syntax, functionality, and implementation, and StackOverflow for CSS styling rules.
